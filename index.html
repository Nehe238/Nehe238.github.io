📋 사지선다 (Multiple Choice)

문제 1

핵심 개념: SIMT

  • GPU는 SIMT (Single Instruction, Multiple Threads) 아키텍처 사용
  • 전통적 SIMD와 달리, 각 thread가 독립적 execution context를 가짐
  • 같은 warp 내에서 branch 발생 시 warp divergence → serial 수행
GPU의 실행 아키텍처 중 "하나의 명령어를 여러 스레드에 동시 적용하되, 각 스레드가 독립적 컨텍스트를 갖는" 방식은?
문제 2

핵심 개념: Warp

  • Warp = CUDA의 기본 실행 단위
  • 하나의 warp는 32개의 thread로 구성
  • 동일한 instruction을 실행, 각 warp는 독립적 PC와 register state 보유
  • Zero context switching overhead: register가 SM 안에 항상 유지됨
CUDA에서 하나의 Warp를 구성하는 스레드의 수는?
문제 3

핵심 개념: Thread 계층 최대 크기

  • Block: 최대 3차원, x·y ≤ 1024, z ≤ 64, 총 스레드 수 ≤ 1024
  • Grid: 최대 3차원, x ≤ 2³¹-1, y·z ≤ 65535
하나의 CUDA Block 안에 담을 수 있는 최대 스레드 수는?
문제 4

핵심 개념: CUDA 전역 Thread ID

  • 1D Grid, 1D Block 기준 전역 스레드 ID:
  • tID = blockIdx.x * blockDim.x + threadIdx.x
1D Grid, 1D Block 구성에서 현재 스레드의 전역(global) thread ID를 올바르게 계산한 식은?
문제 5

핵심 개념: CUDA 메모리 계층 속도

  • Register: <1 cycle (가장 빠름, per-thread)
  • Shared Memory: 1~4 cycles (on-chip, per-block)
  • Global Memory: 400~800 cycles (가장 느림, off-chip)
  • Local Memory: 400~600 cycles (register spilling 시 사용)
CUDA 메모리 중 접근 속도가 가장 빠른 메모리는?
문제 6

핵심 개념: Shared Memory 역할

  • __shared__ 키워드로 선언
  • 같은 Block 내 모든 스레드가 공유 (inter-thread communication)
  • User-managed cache로 global memory 접근 횟수 감소
  • __syncthreads() 필수: 모든 스레드가 로딩 완료 후 연산 시작
CUDA에서 같은 Block 내 스레드 간 데이터를 빠르게 공유하기 위해 사용하는 메모리는?
문제 7

핵심 개념: __syncthreads()

  • Block 내 모든 스레드가 해당 지점에 도달할 때까지 대기 (barrier)
  • 주로 Shared Memory 로딩 완료 확인 후 연산 시작 시 사용
  • Block 범위 동기화이며, Grid 전체 동기화는 별도 Kernel 호출 필요
__syncthreads()의 역할로 가장 적절한 것은?
문제 8

핵심 개념: Warp Divergence

  • 하나의 warp 안에서 서로 다른 branch(if/else)를 따를 때 발생
  • 분기별로 직렬화(serialization)되어 처리 → 성능 저하
  • GPU 알고리즘 설계 시 warp divergence 최소화 필요
Warp Divergence가 발생했을 때 나타나는 현상은?
문제 9

핵심 개념: Coalesced Memory Access

  • Warp(32 threads)가 연속된 메모리 주소에 접근 = Coalesced
  • L2: 32-byte transaction / L1: 128-byte transaction
  • Coalesced + Aligned → 최소 transaction 수 → 최고 성능
  • 행렬에서 col = threadIdx.x로 지정해야 coalesced 접근
Global Memory 성능 최적화를 위한 Coalesced memory access 조건은?
문제 10

핵심 개념: Occupancy

  • Occupancy = (# 활성 warp 수) / (SM의 최대 warp 수)
  • 높은 Occupancy → 메모리 지연 숨기기(latency hiding) 효과 ↑
  • 결정 요소: thread당 register 수, block당 thread 수, shared memory 크기
CUDA Occupancy를 높이기 위한 접근 방법으로 가장 적절하지 않은 것은?
문제 11

핵심 개념: Atomic 함수

  • read-modify-write를 원자적으로 수행 (race condition 방지)
  • atomicAdd(), atomicSub(), atomicMin(), atomicMax()
  • Global 또는 Shared memory에 대해 사용 가능
  • Shared atomic + Global atomic 조합으로 성능 향상 가능
여러 스레드가 동시에 같은 변수에 값을 더하는 Race Condition을 해결하기 위한 올바른 방법은?
문제 12

핵심 개념: CUDA Stream

  • NULL stream: 동기(synchronous) 실행 – 순서 보장
  • Non-NULL stream: 비동기(asynchronous) 실행 – 동시 수행 가능
  • cudaMallocHost() = Pinned memory → 비동기 전송 지원 + 빠름
  • cudaMemcpyAsync() + stream으로 전송과 커널 overlap 가능
CUDA에서 비동기 메모리 전송(cudaMemcpyAsync)을 사용하려면 Host 메모리가 어떤 상태여야 하는가?

✏️ 빈칸채우기 (Fill in the Blank)

정답 입력 후 [확인] 버튼 클릭 | 영어는 대소문자 구분 없음

빈칸 1

핵심: GPU 실행 아키텍처 이름

  • GPU는 SIMD가 아닌 SIMT 아키텍처를 사용한다. (Single Instruction, Multiple Threads)
GPU는 SIMD 방식이 아닌 (Single Instruction, Multiple Threads) 아키텍처를 사용한다.
빈칸 2

핵심: Warp 크기

  • CUDA의 기본 실행 단위인 warp는 32개 스레드로 구성된다.
CUDA에서 기본 실행 단위(Warp)는 개의 스레드로 구성된다.
빈칸 3

핵심: 커널 함수 선언자

  • __global__: Host가 호출, Device에서 실행
  • __device__: Device 내부에서만 호출/실행
  • __host__: Host에서 호출, Host에서 실행 (기본값)
GPU에서 실행되며 Host에서 호출하는 CUDA 커널 함수는 키워드로 선언한다.
빈칸 4

핵심: Device 메모리 할당 API

  • cudaMalloc(void** ptr, size_t size): GPU 메모리 할당
  • cudaFree(void* ptr): GPU 메모리 해제
GPU(Device) 메모리를 할당하기 위해 함수를 사용하고, 해제 시에는 cudaFree()를 사용한다.
빈칸 5

핵심: Shared Memory 선언

  • __shared__ int arr[512]; — 정적 할당
  • extern __shared__ int arr[]; — 동적 할당 (커널 호출 시 크기 지정)
Block 내 스레드가 공유하는 On-chip 메모리는 키워드로 선언한다.
빈칸 6

핵심: Block 내 동기화 함수

  • Shared memory에 데이터를 모두 올린 뒤 연산하기 전에 반드시 호출
  • Block 내 모든 스레드가 이 지점에 도달해야 다음으로 진행
Block 내 모든 스레드가 특정 지점까지 실행되도록 기다리는 함수는 이다.
빈칸 7

핵심: Register Spilling

  • Thread가 사용하는 지역 변수 수 > SM의 register 수 초과 시
  • 초과된 register가 off-chip DRAM인 Local Memory로 넘쳐흐름
  • Local Memory는 400~600 cycles → 성능 급락
스레드가 사용하는 지역 변수가 Register 용량을 초과할 때, 초과분이 느린 off-chip 메모리인 (으)로 넘쳐나는 현상을 Register Spilling이라고 한다.
빈칸 8

핵심: SM (Streaming Multiprocessor)

  • CUDA Core들의 집합 (Fermi 기준 32 CUDA cores per SM)
  • 하나의 Thread Block을 처리하는 단위
  • Register File, Shared Memory, L1 Cache 보유
CUDA GPU에서 하나의 Thread Block을 처리하는 하드웨어 단위를 (Streaming Multiprocessor)라고 한다.
빈칸 9

핵심: Occupancy 공식

  • Occupancy = 활성 warp 수 / 최대 warp 수
  • 높을수록 SM이 더 바쁘게 동작 → 메모리 지연 숨기기 효과
CUDA에서 SM의 최대 warp 수 대비 현재 활성 warp 비율을 (이)라고 한다.
빈칸 10

핵심: Constant Memory

  • __constant__ 키워드 + 전역 범위 선언
  • Read-only, Host가 초기화 (cudaMemcpyToSymbol)
  • DRAM에 상주, 전용 on-chip cache(64KB/SM)로 빠름
읽기 전용(read-only)이며 Host가 초기화하는 CUDA 메모리 타입은 이며, __constant__ 키워드로 선언한다.
빈칸 11

핵심: Pinned Memory

  • = Page-locked memory: swap-out 안 됨
  • cudaMallocHost()로 할당
  • cudaMemcpyAsync() 사용 시 필수 / 전송 속도도 빠름
Host-Device 간 비동기 데이터 전송을 위해 Host 메모리를 swap-out 되지 않는 (Page-locked) 메모리로 할당해야 한다.
빈칸 12

핵심: 멀티 GPU API

  • cudaGetDeviceCount(&count): GPU 개수 조회
  • cudaSetDevice(id): 사용할 GPU 지정
  • cudaGetDeviceProperties(&prop, id): GPU 속성 조회
시스템에 장착된 CUDA GPU의 개수를 조회하는 함수는 이다.
