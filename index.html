Chapter 8. 교착상태 (Deadlock) 핵심 요약

1. 교착상태의 정의

다중 프로그래밍 시스템에서 둘 이상의 프로세스가 서로가 가진 자원을 대기하며 영원히 진행이 멈춰버리는 현상.

2. 교착상태 발생의 4가지 필요조건 (★ 무조건 암기, PPT 용어 고수)

  • 상호배제 (Mutual Exclusion): 한 번에 한 프로세스만 자원을 점유할 수 있음.
  • 점유와 대기 (Hold and Wait): 최소한 하나의 자원을 점유한 채, 다른 프로세스에 할당되어 있는 자원을 추가로 요구하며 대기함.
  • 비선점 (No Preemption): 다른 프로세스에 할당된 자원은 끝날 때까지 강제로 빼앗을 수 없음.
  • 환형대기 (Circular Wait): 대기 프로세스 집합 {P0, P1, ..., Pn}에서 P0는 P1을, P1은 P2를, Pn은 P0를 기다리는 원형 궤도를 구성함.
주의: 이 4가지 조건이 동시에 만족해야만 교착상태가 발생합니다. 하나라도 파괴하면 예방할 수 있습니다.

3. 교착상태 해결 기법 비교

  • 예방 (Prevention): 발생 조건 4가지 중 하나를 제거함. (자원 효율성이 매우 저하됨)
    • 환형대기 제거: 모든 자원에 고유 번호(순서)를 부여하고 순서대로만 요청하게 함. (단점: 자원 유연성 저하, 자원 낭비)
  • 회피 (Avoidance): 자원 요청 시 시스템이 안전 상태를 유지할 수 있을 때만 자원을 할당함.
    • 대표 알고리즘: 은행원 알고리즘 (Banker's Algorithm)
  • 검출 및 복구 (Detection and Recovery): 교착상태 발생을 허용하되 주기적으로 검출 알고리즘을 수행하여 발견 시 복구(프로세스 강제 종료 또는 자원 선점)함.

Chapter 9. 주기억장치 관리 핵심 요약

1. 기억장치 배치 전략 (Placement Strategy)

새로운 프로그램이나 데이터를 주기억장치의 어디에 위치시킬 것인가에 대한 전략.

  • 최초 적합 (First-fit): 가용 공간 리스트의 처음부터 탐색하여 첫 번째로 발견된 충분한 크기의 빈 공간에 할당. (가장 빠름)
  • 최적 적합 (Best-fit): 가용 공간 전체를 탐색하여 요구하는 크기와 가장 차이가 적은(작은 여유 공간을 남기는) 빈 공간에 할당. (단점: 매우 작은 쓰레기 공간이 다수 발생)
  • 최악 적합 (Worst-fit): 가용 공간 전체를 탐색하여 가장 큰 빈 공간에 할당. (남는 공간이 커서 다른 프로세스가 재활용하기 좋음)

2. 단편화 (Fragmentation) 현상

  • 내부 단편화 (Internal Fragmentation): 고정 분할 구조에서, 할당된 작업의 크기가 분할 영역 크기보다 작아서 해당 분할 내부에 사용되지 못하고 낭비되는 빈 공간.
  • 외부 단편화 (External Fragmentation): 가변 분할 구조에서, 대기 중인 작업보다 가용 공간 총량은 충분하지만, 공간들이 잘게 조각나 분산되어 있어 할당하지 못하는 현상.
해결책: 외부 단편화는 조각난 빈 공간을 하나로 모으는 통합(Compaction) 또는 압축(Garbage Collection) 기법으로 해결할 수 있습니다.

Chapter 10. 가상기억장치 구현 (Paging & Segmentation)

1. 페이징 (Paging) 시스템

  • 가상기억장치(프로그램)를 고정된 크기인 페이지(Page)로 분할하고, 주기억장치도 동등한 크기인 페이지 프레임(Page Frame)으로 분할하여 관리.
  • 특징: 외부 단편화는 전혀 발생하지 않으나, 마지막 페이지에서 내부 단편화가 발생할 수 있음.
  • 가상 주소 구조: v = (p, d) (p: 페이지 번호, d: 변위/디스플레이스먼트)
  • 주소 사상 기법(Address Mapping): PMT (Page Map Table, 페이지 사상 테이블)를 참조하여 가상 주소 p를 물리적 프레임 주소 p'로 변환.
    • 직접 사상 (Direct Mapping): PMT가 주기억장치에 존재함. 주소 변환을 위해 주기억장치에 2번 접근하므로 속도가 느림.
    • 연관 사상 (Associative Mapping): 고속 캐시 메모리인 TLB (Translation Lookaside Buffer)라는 전용 하드웨어 연관 기억장치에 PMT를 저장하여 병렬 탐색하므로 매우 빠름.

2. 세그먼테이션 (Segmentation) 시스템

  • 가상기억장치를 논리적 의미 단위인 가변 크기의 세그먼트(Segment)(예: Main 함수, Subroutine, Stack 등)로 분할.
  • 특징: 논리적 단위이므로 공유(Sharing) 및 보안/보호(Protection)에 매우 유리함. 내부 단편화는 발생하지 않으나 주기억장치 배치 해제 과정에서 외부 단편화가 발생함.
  • 가상 주소 구조: v = (s, d) (s: 세그먼트 번호, d: 변위)
  • SMT (Segment Map Table): 각 세그먼트의 주기억장치 상 시작 주소(Base)와 크기인 한계치(Limit) 정보를 관리함.

Chapter 11. 가상기억장치 관리 핵심 요약

1. 페이지 교체 알고리즘 (Page Replacement Algorithm)

주기억장치의 모든 프레임이 사용 중일 때, 새로운 페이지를 적재하기 위해 어떤 페이지를 교체(Victim)할 것인가 결정하는 전략.

  • FIFO (First-In-First-Out): 주기억장치 내에 가장 오래 머물렀던 페이지를 먼저 교체.
    • Belady's Anomaly 현상: 프레임 수를 늘려주었는데도 오히려 페이지 부재(Page Fault) 횟수가 증가하는 모순적 현상 발생.
  • LRU (Least Recently Used): 최근에 가장 오랫동안 사용되지 않은 페이지를 교체. (시간적 지역성 활용, 성능 우수, 실제 널리 쓰임)
  • LFU (Least Frequently Used): 방문/참조 횟수가 가장 적은 페이지를 교체.
  • NUR (Not Used Recently): 최근 미사용 페이지 교체 알고리즘. LRU의 오버헤드를 보완하기 위해 참조 비트(R)와 변형/갱신 비트(M) 2비트 조합으로 교체 우선순위를 결정함.
    • 우선순위 순서 (빨리 교체되는 순서): (R=0, M=0) → (R=0, M=1) → (R=1, M=0) → (R=1, M=1)

2. 스래싱 (Thrashing) 현상과 방지 대책

  • 스래싱의 정의: 프로세스가 실행되는 시간보다 페이지 부재(Page Fault)가 발생하여 페이지를 교체하는 시간이 더 많아져서 CPU 이용률(Utilization)이 급격히 떨어지는 현상.
  • 워킹셋 (Working Set) 알고리즘: 프로세스가 일정 시간 동안 자주 참조하는 페이지들의 집합을 주기억장치에 항상 상주시켜 스래싱을 방지함.
  • PFF (Page Fault Frequency) 알고리즘: 페이지 부재율의 상한(Upper Bound)과 하한(Lower Bound)을 설정하여, 상한을 넘으면 프레임을 추가 할당하고, 하한 미만으로 떨어지면 프레임을 회수하는 기법.

Chapter 8[서술형] 교착상태(Deadlock)가 발생하기 위해 동시 만족해야 하는 필요조건 4가지를 쓰고 각각 간략히 서술하시오.
모범 답안:
  1. 상호배제(Mutual Exclusion): 한 자원은 한 번에 한 프로세스만 사용할 수 있어야 한다.
  2. 점유와 대기(Hold and Wait): 프로세스가 최소한 하나의 자원을 할당받은 상태에서 다른 프로세스가 사용 중인 자원을 추가로 요구하며 대기해야 한다.
  3. 비선점(No Preemption): 다른 프로세스에 할당된 자원은 해당 프로세스가 자발적으로 반납하기 전까지 강제로 빼앗을 수 없다.
  4. 환형대기(Circular Wait): 대기 상태인 프로세스들이 원형 고리 구조를 이루어 앞의 프로세스가 뒤의 프로세스의 자원을 대기하는 형태를 가져야 한다.
Chapter 8[서술형] 교착상태 예방(Prevention) 기법 중 "환형대기" 조건을 배제하는 구체적인 기법명과 이 기법이 초래할 수 있는 문제점(단점)에 대해 기술하시오.
모범 답안:
  • 구체적 기법: 시스템 내의 모든 자원에 고유한 선형 순서(번호)를 부여하고, 각 프로세스는 현재 점유한 자원보다 오직 높은 번호(순서)의 자원만을 요청할 수 있도록 제한하는 기법이다.
  • 단점/문제점: 실제 프로그램 실행 중 자원이 필요한 시점과 상관없이 번호 순서대로만 자원을 획득해야 하므로 자원의 효율적 이용이 저하(자원 낭비)되며, 새로운 자원이 추가될 때 기존 자원들의 일련번호를 전면 재수정해야 하므로 유연성이 크게 떨어진다.
Chapter 9[코드 완성] 최초 적합(First-fit) 메모리 할당 알고리즘을 구현한 아래 C언어 기반 수도코드에서 빈칸 (1)과 (2)에 들어갈 적절한 코드를 작성하시오.
void allocateFirstFit(int request_size) {
    for (int i = 0; i < block_count; i++) {
        if (free_list[i].is_free == true && free_list[i].size >= request_size) {
            
            // (1) 선택된 메모리 블록에 프로세스 자원을 할당 및 남는 영역 처리 수행
                  빈칸 (1)      ; 
            
            free_list[i].is_free = false;
            
            // (2) 첫 번째 적합한 블록을 찾았으므로 탐색 루프를 탈출해야 함
                  빈칸 (2)      ; 
        }
    }
}
모범 답안:
  • 빈칸 (1): splitBlock(i, request_size) 또는 free_list[i].size = request_size (※ 메모리 분할 또는 할당 로직 명시)
  • 빈칸 (2): break
해설: First-fit은 순차 검색을 하다가 요청한 크기(request_size)를 만족하는 최초의 블록을 찾으면 할당을 진행한 직후 바로 break 문을 통해 반복 루프를 빠져나와야 다음 블록들을 불필요하게 검사하지 않습니다.
Chapter 9[서술형] 메모리 단편화 현상 중 내부 단편화(Internal Fragmentation)와 외부 단편화(External Fragmentation)의 개념을 명확히 비교하여 서술하시오.
모범 답안:
  • 내부 단편화: 주기억장치를 고정 분할했을 때, 분할된 구역에 할당된 프로세스의 크기가 구역 크기보다 작아서, 해당 분할 영역 내부에서 사용되지 못하고 낭비되는 빈 메모리 공간이다.
  • 外部 단편화: 주기억장치를 가변 분할했을 때, 메모리 상의 빈 공간들의 총합은 프로세스가 요구하는 크기보다 충분히 크지만, 빈 공간들이 여러 군데로 미세하게 조각나 분산되어 있어 프로세스를 적재하지 못하고 낭비되는 현상이다.
Chapter 10[그림 구현 및 메커니즘] 세그먼테이션(Segmentation) 시스템에서 가상 주소 v = (s, d)가 SMT(세그먼트 사상 테이블)를 거쳐 실주소(물리 주소) r로 변환되는 주소 사상 과정을 논리적 흐름도 다이어그램으로 도식화하고 변환 원리를 설명하시오.
1. 주소 사상 메커니즘 흐름도 (도식화 모델):
가상주소 v = (s, d)
s: 세그먼트 번호 / d: 변위
↓
SMT (세그먼트 테이블 인덱싱 s)
Segment (s)변위 한계치 (Limit)시작 주소 (Base)
sSegment LimitBase Address (b)
↓ [조건 검사: d > Limit 이면 Trap 발생]
연산기: Base (b) + 변위 (d)
↓
실주소 r = b + d
2. 변환 원리 서술형 답안:
  1. 프로세스가 가상 주소 v = (s, d)를 참조하면 세그먼트 번호 s를 인덱스로 사용하여 SMT(세그먼트 사상 테이블)로 이동한다.
  2. SMT에서 해당 세그먼트의 시작 주소(Base, b)와 한계치(Limit) 정보를 획득한다.
  3. 변위 d가 한계치(Limit)를 초과하는지 검사하여, d > Limit이면 한계 초과 에러 인터럽트(Trap)를 발생시키고 차단한다.
  4. 검사를 통과하면 시작 주소 b에 변위 d를 더하여 실제 주기억장치 주소인 r = b + d를 계산해 낸다.
Chapter 11[서술형] 가상기억장치 관리 중 스래싱(Thrashing)의 정의를 명확히 내리고, 운영체제에서 이를 방지하기 위해 사용하는 대표적인 알고리즘인 워킹셋(Working Set) 기법의 작동 원리를 쓰시오.
모범 답안:
  • 스래싱(Thrashing)의 정의: 다중 프로그래밍 시스템에서 실행 중인 프로세스가 실행 시간보다 페이지 부재(Page Fault) 빈도가 과도하게 높아져 **새로운 페이지를 교체하고 적재하는 시간이 더 많아짐으로써 CPU 이용률이 급격히 저하되는 현상**이다.
  • 워킹셋(Working Set) 작동 원리: 지역성(Locality) 이론을 바탕으로, 프로세스가 일정 시간(윈도우 크기 Δ) 동안 **자주 집중적으로 참조하는 페이지들의 집합(Working Set)을 식별하고 이들을 주기억장치에 동시에 상주**시킴으로써 페이지 부재를 최소화하고 스래싱을 예방한다.
Chapter 11[서술형] 페이지 교체 알고리즘 중 NUR(Not Used Recently)의 핵심 원리와 교체 우선순위를 결정하기 위해 사용되는 비트들의 종류 및 역할을 설명하시오.
모범 답안:
  • 핵심 원리: LRU 알고리즘의 단점인 시간 기록 오버헤드를 줄이기 위해 하드웨어적인 비트만을 이용하여 최근에 사용되지 않은 페이지를 대상(Victim)으로 선택하는 효율적 기법이다.
  • 사용 비트 종류 및 역할:
    1. 참조 비트 (Reference Bit, R): 페이지가 참조(읽기/쓰기)될 때 하드웨어에 의해 1로 세팅된다.
    2. 변형 비트 (Modified Bit, M): 페이지의 내용이 수정(Write)되었을 때 1로 세팅된다.
  • 교체 우선순위: 참조 비트가 0인 것이 우선 교체되며, 둘 다 같으면 변형 비트가 0인 것(디스크에 다시 쓸 필요가 없는 페이지)이 먼저 교체된다. 순서는 (R=0, M=0) → (R=0, M=1) → (R=1, M=0) → (R=1, M=1) 이다.
