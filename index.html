8장

교착상태

Deadlock

교착상태의 의미

교착상태(deadlock state)프로세스가 전혀 발생할 가능성이 없는 사건을 기다리는 경우, 그 프로세스는 교착상태에 놓여 있다고 한다. 교착상태에 놓인 프로세스가 하나 이상 있으면 그 시스템은 교착상태에 있다고 한다.

→ 무기한 연기(indefinite postponement) 상태와 구분: 무기한 연기는 자원이 할당될 수는 있으나 계속 밀리는 것, 교착은 외부 개입 없이 절대 풀리지 않음.

자원의 분류

분류 기준종류
선점 가능성선점가능 자원 / 선점불가능 자원
할당 방식전체 할당식 자원 / 분할 할당식 자원
할당 형태배타적 할당 자원 / 공유식 할당 자원
자원의 속성순차적 재사용(SR) 자원 / 소비성(CR) 자원

SR(Serially Reusable) = 없어지지 않고 영구히 존재 (프로세서·메모리·버퍼·테이프·프로그램)   CR(Consumable) = 사용 후 사라짐 (시그널·메시지).

교착상태 모델 대상 자원 = 선점불가능 · 배타적 할당 · 순차적 재사용. (소비성 자원 대상 모델도 별도 존재)

자원할당 그래프 모형

프로세스 노드 Pi  ·  자원 노드 Rj

에지의미
Rj → Pi해당 자원이 프로세스에게 할당됨
Pi → Rj프로세스가 해당 자원을 요청, 대기중

교착상태 발생의 4가지 필요조건 (necessary conditions)

다음 4가지 조건이 모두 만족될 때 교착 발생.

#필요조건
①자원에 대한 배타적 사용 (exclusive use)
②선점불가능 자원(nonpreemptible resources)의 존재
③프로세스에 대한 자원의 부분 할당 (partial allocation)
④환형대기 상태(circular wait condition)의 발생

교착상태 해결 기법

기법내용
예방(prevention)4가지 필요조건 중 하나를 제거 — 배타적 사용 배제(공유) / 선점 기능 지원 / 자원의 전체 할당 / 환형 대기 제거. 자원의 심각한 낭비, 현실적으로 어려움
회피(avoidance)시스템을 항상 안전상태로만 유지 — Dijkstra's Banker's algorithm, Habermann's algorithm
검출(detection)발생을 허용하되 그래프 단순화로 탐지
복구(recovery)탐지 후 프로세스 종료 또는 자원 선점
안전상태(safe state)안전 순서(safe sequence)가 하나 이상 존재하는 상태. 존재하지 않으면 비안전 상태(unsafe state).

9장

주기억장치 관리

Main Memory Management

주기억장치 구성 정책이 정하는 것

동시에 할당받을 수 있는 프로세스의 수 · 각 프로세스에게 할당되는 양 · 분할 방법 · 분할영역의 교체 가능성 · 할당 영역의 연속성.

분할 방법

분할설명단편화
고정 분할(정적 분할)초기의 분할 형태를 이후 변형하지 않음내부 단편화
가변 분할(동적 분할)초기의 분할 형태를 필요에 따라 변형시킴외부 단편화

교체 가능 정책 = 기억장소 교체(storage swapping). 사용 시 코드가 재배치 가능(relocatable)해야 함.   할당 영역: 연속 / 비연속.

주기억장치 관리 기법 (4)

기법결정 사항
호출 기법(fetch strategy)새 프로세스에 대한 주기억장치 할당 시기 결정
배치 기법(placement strategy)할당받을 프로세스에게 할당 공간의 위치 결정
교체 기법(replacement strategy)공간 부족 시 어느 프로세스 공간을 선점할지 결정
할당 기법(allocation strategy)새 프로세스에 대한 공간 할당량 결정

고정 분할 / 가변 분할 다중프로그래밍

고정 분할 다중프로그래밍(FPM)사용자 공간을 미리 여러 영역으로 분할, 영역당 하나의 프로그램 → 내부 단편화. (재배치 번역 / 재배치 로더 사용)
가변 분할 다중프로그래밍(VPM)프로세스 크기만큼 연속 공간 할당 → 영역 내 단편화 없음, 외부 단편화 발생.
중첩 구조(overlay structure)프로그램이 할당 영역보다 클 때 일부만 번갈아 적재.
경계 레지스터(boundary register)커널·다른 영역 침범 방지를 위해 경계 주소 저장. 다중 분할은 여러 개 사용.

배치 전략 (가변 분할, 4가지)

전략특징
최초 적합(first-fit)처음 발견한 충분한 공간에 배치
최적 적합(best-fit)크기가 가장 근접한(작은) 공간 — 작은 조각 많이 남음
최악 적합(worst-fit)가장 큰 공간 — 단편화 극소화 의도
순환 최초 적합(next-fit)최초 적합과 유사, 직전 위치부터 탐색

단편화 해결

내부 단편화(internal fragmentation)할당된 영역 안에서 사용되지 않고 남는 공간.
외부 단편화(external fragmentation)빈 공간이 흩어져 있어 연속 공간으로 활용하지 못하는 낭비.
인접 공간 통합(coalescing holes)인접한 빈 분할 영역들을 하나로 통합.
기억장소 통합(storage compaction)모든 빈 분할 영역들을 하나로 모아 큰 연속 공간 확보.

10장

가상기억장치 구성

Virtual Memory Organization

프로그램의 블럭 분할 방식

분할 방식시스템
같은 크기의 블럭페이징 시스템
다른 크기의 논리적 단위세그먼테이션 시스템
둘의 합성페이징/세그먼테이션 혼합

주소 사상(address mapping)

주소 사상 함수 f()가상 주소를 실 주소로 변환하는 작업을 실행 중에 동적으로 수행.
블럭 사상(block mapping)블럭 단위로 사상 정보를 블럭 사상 테이블(BMT)에 기록·사용. 블럭이 클수록 테이블 작아지고 사상 빠름. 존재 비트 0 → page fault.

페이징 시스템(paging system)

페이지 / 페이지 프레임페이지 = 분할된 블럭. 페이지 프레임 = 주기억장치 분할 영역(페이지 크기와 동일).

가상 주소 v = (p, d)   사상 테이블: 페이지 사상 테이블(PMT) — 엔트리: 존재 비트 · 보조기억장치 주소 · 페이지 프레임 번호.

논리적 구분 없음 → 공유·보호에 불리.

주소 사상 기법 (3)

기법특징
직접 사상(direct mapping)PMT를 주기억장치에 두고 조회 — 느림(메모리 2회 접근)
연관 사상(associative mapping)연관(고속) 기억장치(TLB)로 병렬 조회 — 빠름, 하드웨어 비쌈
혼합 사상(combined direct/associative)직접+연관 혼합 — 하드웨어 비용↓, 연관 사상의 장점 취함

세그먼테이션 시스템(segmentation system)

서로 다른 크기의 논리적 단위로 분할. 가상 주소 v = (s, d). 사상 테이블: 세그먼트 사상 테이블(SMT : Segment Map Table). 논리 단위라 공유·보호에 유리, 외부 단편화 발생.

페이징/세그먼테이션 혼합

가상 주소 v = (s, p, d) → SMT에서 s로 PMT 위치 → PMT에서 p로 프레임 → 프레임 + d = 실주소.

페이지 공유순수 코드(재진입 가능 코드)일 경우에만 공유 가능.

  • PMT 엔트리 3개: 존재 비트 · 보조기억장치 주소 · 페이지 프레임 번호
  • 존재 비트 0 → 페이지 부재(page fault)
  • 사상 기법 3: 직접(느림) · 연관(TLB, 빠름) · 혼합(절충)
  • 페이징: 공유·보호 불리  /  세그먼테이션: 공유·보호 유리
11장

가상기억장치 관리

Virtual Memory Management

가상기억장치 관리 기법 (6)

기법역할
할당 기법(allocation)각 프로세스에게 할당할 주기억장치 양 결정
호출 기법(fetch)특정 페이지를 언제 적재할지 결정
배치 기법(placement)적재 위치 결정 (최적 적합 등)
교체 기법(replacement)어느 페이지를 내보낼지 결정
소거 기법(cleaning)변경된 페이지를 디스크에 기입
부하 조정 기법(load control)다중프로그래밍 정도 조절

목표 = 페이지 부재 발생율(page fault rate) F() 최소화. 페이지 참조 스트링으로 분석.

할당 기법

기법내용
고정 할당(fixed allocation)메모리 할당량을 실행 동안 고정
가변 할당(variable allocation)메모리 할당량을 실행 동안 변화

호출 기법

기법내용
요구 호출(demand fetch, demand paging)실제 참조한 페이지만 적재 — 부재 시 오버헤드. 대부분 사용
예측 호출(anticipatory fetch, prepaging)참조 전에 미리 적재 — 예측 오버헤드

참조 비트 · 갱신 비트

참조 비트(reference/use bits)페이지 프레임의 최근 참조 여부. 주기마다 0으로 재설정, 참조되면 1.
갱신 비트(update/modify/dirty bits)페이지 프레임 내용의 변경 여부 기록.

지역성(Locality)

종류의미
시간 지역성(temporal locality)최근 참조한 페이지가 곧 다시 참조됨
공간 지역성(spatial locality)참조한 곳 근처가 곧 참조됨

교체 기법

고정 할당 기반 (8가지)

알고리즘내용
MIN(OPT, B0)앞으로 가장 늦게 참조될 페이지 교체 — 최적이나 실현 곤란
무작위(Random)임의로 교체 페이지 선택
FIFO가장 먼저 적재된 페이지 교체 — FIFO의 변칙(Belady's anomaly)
LRU가장 오래 참조되지 않은 페이지 — 지역성 이용
LFU참조 횟수가 가장 적은 페이지
NUR참조 비트·갱신 비트 검사 — 적은 오버헤드로 LRU에 근사
클럭(Clock)참조 비트 사용, 포인터를 시계 방향으로 이동
Second chanceFIFO + 참조 비트로 한 번 더 기회 부여

가변 할당 기반 (3가지)

알고리즘내용
VMIN(Variable MIN)가변 할당 기반의 최적 기법
WS(Working Set)최근 일정 시간(window) 동안 참조한 페이지 집합을 주기억장치에 유지
PFF(Page Fault Frequency)페이지 부재 빈도에 따라 할당량 조절

소거 기법

기법내용
요구 소거(demand cleaning)교체될 때 디스크에 기입 — 대부분 사용
예측 소거(anticipatory cleaning, precleaning)더 변경 안 될 듯할 때 미리 기입

부하 조정 기법(load control)

다중프로그래밍 정도를 조절. 저부하 = 자원 낭비, 고부하 = 자원 경쟁 → 스래싱(thrashing). 스래싱 = 프레임 부족으로 페이지 교체만 반복하며 처리량 급락. 부하 줄여 해결.

정의 — 문장 그대로 외울 것

교착상태(deadlock state)
"프로세스가 전혀 발생할 가능성이 없는 사건을 기다리는 경우, 그 프로세스는 교착상태에 놓여 있다고 한다."
안전상태(safe state)
"안전 순서(safe sequence)가 하나 이상 존재하는 상태."
페이지 부재(page fault)
"존재 비트가 0, 즉 접근할 페이지가 주기억장치에 없는 상황."
시간 지역성
"최근 참조한 페이지가 곧 다시 참조됨."
공간 지역성
"참조한 곳 근처가 곧 참조됨."
스래싱(thrashing)
"고부하로 프레임 부족 → 페이지 교체만 반복 → 처리량 급락."
FIFO의 변칙(Belady's anomaly)
"FIFO에서 프레임 수를 늘렸는데 오히려 페이지 부재가 증가하는 현상."
