8

교착상태 (Deadlock)

교착상태의 정의 PPT 8 - 슬라이드 2, 3

  • 대기 상태(blocked/asleep state): 프로세스가 특정 사건의 발생이나 자원 할당을 기다리는 상태
  • 교착상태(deadlock state): 프로세스가 전혀 발생할 가능성이 없는 사건을 기다리는 상태
  • 교착상태에 놓인 프로세스가 하나 이상 있으면 "시스템이 교착상태에 있다"고 함
  • 교착상태와 무기한 연기(indefinite postponement)는 구분 필요

자원의 분류 PPT 8 - 슬라이드 4~9

선점가능성에 의한 분류

  • 선점가능 자원: 선점 후 복귀시 문제 없음 (예: 프로세서, 메모리)
  • 선점불가능 자원: 선점시 해당 프로세스에 영향 (예: 자기테이프 장치)

할당 방식에 따른 분류

  • 전체 할당식: 항상 자원 전체를 할당 (프로세서, 자기테이프 등)
  • 분할 할당식: 분할된 조각 단위로 할당 (메모리 등)

할당 형태에 따른 분류

  • 배타적 할당: 여러 프로세스가 동시에 사용 불가 (프로세서, 메모리, 버퍼 등)
  • 공유식 할당: 여러 프로세스가 동시에 사용 가능 (프로그램, 공유 데이터)

자원의 속성에 따른 분류

  • SR(Serially Reusable): 없어지지 않고 영구적으로 존재 (프로세서, 메모리, 프로그램 등)
  • CR(Consumable Resources): 사용 후 사라짐 (시그널, 메시지 등)
교착상태 모델의 대상 자원: 선점불가능 + 배타적 할당 + 순차적 재사용(SR) 자원

교착상태 발생 원인 - 4가지 필요조건 PPT 8 - 슬라이드 14

다음 4가지가 모두 동시에 성립할 때 교착상태 발생
  1. 자원에 대한 배타적 사용 (Exclusive use): 한 번에 하나의 프로세스만 사용
  2. 선점불가능 자원의 존재 (Non-preemptible): 강제로 빼앗을 수 없음
  3. 부분 할당 (Partial allocation): 자원을 일부만 할당받은 상태에서 추가 자원 요청
  4. 환형 대기 (Circular wait): 프로세스들이 원형으로 서로의 자원을 요청

교착상태 해결 기법 개요 PPT 8 - 슬라이드 15

  • 예방(Prevention): 4가지 필요조건 중 하나를 제거
  • 회피(Avoidance): 시스템 상태를 감시하여 항상 안전상태 유지
  • 검출(Detection) + 복구(Recovery): 교착상태 발생 후 감지하고 제거

교착상태 예방 기법 PPT 8 - 슬라이드 16~20

제거 조건방법단점
배타적 사용 배제모든 자원을 공유 가능하게 함실제로 불가능
선점 기능 지원자원 요청 불가시 기존 자원 전부 반납자원 낭비 심함, 비실현적
부분 할당 배제자원 전체 할당 기법: 실행 전 모든 자원을 미리 할당자원 낭비, 무기한 연기 가능
환형 대기 배제자원에 순서를 부여, 한 방향으로만 요청 가능자원 낭비
대부분의 경우 자원의 심각한 낭비를 초래하며 현실적으로 사용하기 어려움

교착상태 회피 기법 PPT 8 - 슬라이드 21~35

  • 항상 시스템 상태를 감시, 교착상태 전이 가능성이 있으면 자원 할당 유보
  • 안전상태(safe state): 모든 프로세스를 유한 시간 내에 종료시킬 수 있는 상태 (safe sequence 존재)
  • 비안전상태(unsafe state): 교착상태로의 전이를 피하지 못할 가능성 존재 (반드시 교착상태 발생을 의미하지는 않음)

Dijkstra's Banker's Algorithm 슬라이드 25~30

  • 자원형이 한 가지만 존재하는 경우에 적용
  • 자원 요청 시 할당 후 상태가 안전상태이면 승인, 비안전상태면 거절

Habermann's Algorithm 슬라이드 31~35

  • Dijkstra 알고리즘의 확장: 여러 종류의 자원형 존재 시 적용
  • 마찬가지로 자원 할당 후 안전상태 여부 확인 후 승인/거절
가정: 프로세스 수 고정, 자원 수 고정, 각 프로세스의 최대 요구량이 미리 알려져야 함 → 비현실적(not practical)

교착상태 검출 기법 PPT 8 - 슬라이드 36~45

  • 사전 처리 없이, 주기적으로 또는 필요한 경우 교착상태 여부 확인
  • 자원 할당 그래프(resource allocation graph) 사용
    • 요구 에지: Pi → Rj (프로세스가 자원 요청)
    • 할당 에지: Rj → Pi (자원이 프로세스에 할당됨)
  • 그래프 단순화(graph reduction) 기법:
    • unblocked 프로세스(현재 요구한 자원을 모두 할당받을 수 있는 프로세스)를 찾아 인접 에지 제거 반복
    • 완전 단순화(completely reduced): 모든 에지 제거 → 교착상태 없음
    • 단순화 불가(irreducible): 남은 에지 존재 → 교착상태 존재
  • 특수 경우: single-unit resource → 사이클 검사, single-unit request → knot 검사
회피 vs 검출 차이
회피: 최악의 경우(worst case) 고려
검출: 최선의 경우(most favorable case) 고려, 현재 상태만 검사

교착상태 복구 기법 PPT 8 - 슬라이드 47~51

프로세스 종료(Process Termination) 기법

  • 교착상태 프로세스 중 일부를 강제 종료 후 재시작
  • 종료 비용 산출 요소: 우선순위, 종류, 현재 실행시간, 남은 실행시간, 회계 비용
  • lowest-termination-cost first: 단순, 오버헤드 적음, 불필요한 종료 가능
  • minimum cost recovery: 최소 비용으로 교착상태 제거, 선정 과정 복잡

자원 선점(Resource Preemption) 기법

  • 선점할 자원 선정 → 해당 자원 보유 프로세스 강제 종료 후 재실행
  • 교착상태가 아닌 프로세스도 종료 대상이 될 수 있음

검사점-재시작(Checkpoint-Restart) 기법

  • 검사점(checkpoint)에서 프로세스의 context를 저장
  • 강제 종료 후 가장 최근 검사점부터 재실행
9

주기억장치 관리

기억장치 계층구조 PPT 9 - 슬라이드 2, 3

  • 프로세서 레지스터 → 캐쉬 메모리 → 주기억장치 → 보조기억장치 (위로 갈수록 빠르고 비쌈)
  • 블럭(block): 보조기억장치 ↔ 주기억장치 데이터 전송 단위
  • 워드(word): 주기억장치 ↔ 프로세서 레지스터 데이터 전송 단위
  • OS는 주기억장치와 보조기억장치를 관리 (레지스터, 캐쉬는 하드웨어가 제어)

주기억장치 관리 기법 PPT 9 - 슬라이드 10

  • 호출 기법: 언제 주기억장치에 적재할지 결정
  • 배치 기법: 어느 위치에 적재할지 결정 (first-fit, best-fit, worst-fit, next-fit)
  • 교체 기법: 공간 부족시 어느 프로세스 공간을 선점할지 결정
  • 할당 기법: 새 프로세스에 얼마의 공간을 할당할지 결정

단일프로그래밍 시스템 PPT 9 - 슬라이드 11~14

  • 항상 하나의 프로세스만 존재
  • 문제점:
    1. 프로그램이 가용 공간보다 클 경우 → 중첩 구조(overlay structure)로 해결
    2. 커널 보호 필요 → 경계 레지스터(boundary register)로 해결
    3. 자원 낭비/성능 저하 → 다중프로그래밍으로 해결

고정 분할 다중프로그래밍 (FPM) PPT 9 - 슬라이드 15~21

  • 주기억장치를 미리 여러 영역으로 분할, 각 분할 영역에 하나의 프로그램 적재
  • 분할영역 수 = k이면 최대 다중프로그래밍 정도 k
  • 장점: OS 관리 용이, 오버헤드 작음
  • 단점:
    • 내부 단편화(internal fragmentation): 분할 영역 내에서 낭비되는 공간
    • 외부 단편화(external fragmentation): 크기 문제로 분할 영역 전체가 낭비
  • 보호: 여러 개의 경계 레지스터 사용

가변 분할 다중프로그래밍 (VPM) PPT 9 - 슬라이드 22~36

  • 초기에 사용자 공간 전체를 하나의 분할 영역으로 설정, 프로세스 활동에 따라 동적 변화
  • 분할 영역 내 단편화 없음, 관리 오버헤드 증가

배치 기법 슬라이드 31~32

기법설명특징
First-fit처음부터 검사, 충분한 첫 번째 영역단순, 오버헤드 적음
Best-fit크기가 맞는 가장 작은 영역큰 빈 공간 확보, 외부 단편화 발생
Worst-fit가장 큰 영역에 배치단편화 극소화 가능, 대용량 빈 공간 확보 불가
Next-fit직전 검사 마지막 위치부터 탐색각 영역 사용 빈도 균등화

공간 관리 기법 슬라이드 33~36

  • 인접 공간 통합(coalescing holes): 인접한 빈 분할 영역 합침
  • 기억장소 통합(storage compaction): 모든 빈 분할 영역을 하나로 통합, 모든 프로세스 재배치 필요 → 오버헤드 매우 큼
  • 기억장소 교체(storage swapping): 실행 종료 안 된 프로세스의 기억장소를 중간에 반납받는 기법
10

가상기억장치 구성

가상기억장치 개념 PPT 10 - 슬라이드 2

  • 사용자 프로그램을 여러 블럭으로 분할, 실행 시 필요한 블럭만 비연속적으로 주기억장치에 적재
  • 분류: 페이징 시스템, 세그먼테이션 시스템, 혼합 기법

주소 사상 PPT 10 - 슬라이드 3~11

  • 가상 주소(virtual address) = 상대 주소: 프로그램 시작점을 0으로 가정
  • 실 주소(real address) = 절대 주소
  • 주소 사상 함수 f(): 가상 주소 v를 실 주소 r로 변환 (f: V → R, r = f(v))
  • 인위적 연속성: 프로그램 전체가 연속으로 적재된 것처럼 보이게 하는 개념
  • 블럭 사상: 가상 주소 v = (b, d), 블럭 번호 + 변위(offset)

페이징 시스템 PPT 10 - 슬라이드 12~31

  • 정의: 프로그램을 같은 크기의 블럭(페이지)으로 분할
  • 페이지(page): 실행 프로그램의 분할 블럭
  • 페이지 프레임(page frame): 주기억장치의 분할 영역 (페이지와 같은 크기)
  • 특성: 단순하고 효율적, 논리적 분할 개념 없음, 프로그램 공유/보호에 복잡한 문제 발생

주소 사상: v = (p, d) 슬라이드 15~25

  • PMT(Page Map Table) 이용: 페이지 번호, 존재 비트, 보조기억장치 주소, 페이지 프레임 번호
사상 기법방법특징
직접 사상PMT를 주기억장치 커널 공간에 저장주기억장치 접근 2배 증가, 성능 저하
연관 사상PMT를 연관 기억장치(associative memory)에 적재, 병렬 탐색하드웨어 비용 큼
혼합 사상소규모 연관 기억장치 + 주기억장치에 전체 PMT지역성 활용, 비용과 성능의 타협
페이지 부재(page fault): 프로세스가 접근하는 페이지가 주기억장치에 없는 경우
→ 직접 사상 알고리즘의 존재 비트가 0인 경우 발생

페이지 공유 슬라이드 28~31

  • 프로시져 페이지(순수 코드, 재진입가능 코드)만 공유 가능
  • 공유 페이지는 각 프로세스의 PMT에서 같은 엔트리 번호로 저장해야 분기 주소 오류 방지

세그먼테이션 시스템 PPT 10 - 슬라이드 32~42

  • 정의: 프로그램을 논리적인 의미를 가진 서로 다른 크기의 블럭(세그먼트)으로 분할
  • 특징: 주기억장치 영역 미리 분할 불가, 빈 공간 찾아 할당(VPM과 유사), 공유·보호 쉬움, 관리 오버헤드 증가

주소 사상: v = (s, d) 슬라이드 34~38

  • SMT(Segment Map Table): 세그먼트 번호, 존재 비트, 보조기억장치 주소, 세그먼트 길이, 보호 비트, 세그먼트 주소
  • 직접 사상 알고리즘 처리 순서: 존재 비트 확인 → 변위 > 세그먼트 길이 확인(overflow) → 보호 비트 확인 → 실 주소 계산

접근 제어 슬라이드 39

  • SMT의 보호 비트 필드 사용
  • R(Read), W(Write), X(eXecute), A(Append) 권한

페이징/세그먼테이션 혼합 기법 PPT 10 - 슬라이드 43~55

  • 두 기법의 장점 결합: 논리적 분할(세그먼테이션) + 단순 관리(페이징)
  • 분할: 사용자 프로그램 → 세그먼트 → 각 세그먼트를 다시 페이지로 분할
  • 가상 주소: v = (s, p, d)
  • SMT의 존재 비트 없음 (페이지 단위로 적재되기 때문)
  • 단점: 사상 테이블 개수·전체 크기 증가, 주소 사상 시간 증가, 직접 사상시 성능 1/3 수준 저하
기법장점단점
페이징단순, 효율적 주소 사상논리적 개념 없음, 공유 문제 복잡
세그먼테이션논리적 분할, 공유 쉬움관리 오버헤드 큼
혼합 기법단순성·효율성·공유 모두 해결사상 과정 복잡, 테이블 크기 증가
11

가상기억장치 관리

비용 모델과 지역성 PPT 11 - 슬라이드 4, 5, 20

  • 비용 모델: 페이지 부재(page fault) 발생 횟수 최소화를 목적으로 설계
  • 페이지 참조 스트링(ω): 접근한 페이지 번호를 접근 순서대로 나열
  • 페이지 부재율 F(ω) = 페이지 부재 발생 횟수 / |ω|
지역성(Locality): 프로그램 내 일부 명령이 집중적으로 실행되는 현상
- 시간 지역성: 한 번 실행/접근된 부분이 이후 재실행/재접근될 가능성 높음
- 공간 지역성: 직전 실행/접근 부분의 인근 부분이 이후 실행/접근될 가능성 높음

하드웨어 요소 PPT 11 - 슬라이드 6~11

  • 참조 비트(reference bit): 페이지 프레임이 최근 참조되었는지 기록, 주기적으로 0으로 재설정
  • 갱신 비트(update/dirty bit): 페이지 내용이 변경되었는지 기록, 주기적 재설정 없음 → write-back 필요

가상기억장치 관리 기법 분류 PPT 11 - 슬라이드 3, 12~18

  • 할당 기법: 각 프로세스에 할당할 주기억장치의 양 결정 (고정/가변 할당)
  • 호출 기법: 언제 적재할지 (요구 호출 vs 예측 호출, 실제로는 요구 호출 주로 사용)
  • 배치 기법: 어느 페이지 프레임에 적재할지 (페이징 시스템에서는 불필요)
  • 교체 기법: 빈 프레임 없을 때 어느 페이지를 교체할지
  • 소거 기법: 변경된 페이지를 언제 디스크에 저장할지 (요구 소거 vs 예측 소거)
  • 부하 조정 기법: 다중프로그래밍 정도 조절 (thrashing 방지)
스래싱(Thrashing): 페이지 부재가 과도하게 발생하는 현상 (고부하 상태)
다중프로그래밍 정도가 너무 높아지면 발생, plateau 영역에서 적절히 유지해야 함

고정 할당 기반 교체 기법 PPT 11 - 슬라이드 23~50

알고리즘선정 기준특징
MIN (OPT, B₀)이후 가장 오래 참조되지 않을 페이지최적 기법(proved), 미래 참조 스트링이 필요하여 실현 불가능. 성능 평가 도구로 사용
Random임의 선정오버헤드 적음, 최적화 시도 없음
FIFO가장 먼저 적재된 페이지구현 단순. FIFO Anomaly(Belady's anomaly): 프레임 수 증가 시 페이지 부재가 오히려 증가하는 경우 발생
LRU가장 오래 참조되지 않은 페이지지역성 기반, MIN에 근사. 참조 시간 기록 오버헤드 큼. 실제 가장 많이 사용. 대형 반복 구조에서 성능 급격 저하
LFU참조 횟수 가장 적은 페이지지역성 이용, LRU 오버헤드 감소. 최근 적재된 페이지가 교체될 수 있음
NUR참조 비트(r), 갱신 비트(m) 순서로 교체: (0,0)→(0,1)→(1,0)→(1,1)적은 오버헤드로 LRU 성능 근사
Clock환형 리스트 포인터 이동, 참조 비트=0인 페이지 교체IBM VM/370 사용. FIFO와 LRU의 혼합적 특성
Second ChanceClock 알고리즘 + 갱신 비트 함께 고려, (r,m): (0,0)→교체, (0,1)→소거 후 진행, (1,0)→(0,0)후 진행, (1,1)→(0,1)후 진행Clock의 확장
LRU 구현 방법
1. 카운터(counter) 방법: 각 프로세스 PMT에 카운트 필드, 참조 시마다 프로세서 클럭 값 기록
2. 스택(stack) 방법: 최근 참조 순서대로 페이지 번호 유지, 중간 삭제 가능한 스택 사용

가변 할당 기반 교체 기법 PPT 11 - 슬라이드 58~77

Working Set (WS) 기법 슬라이드 58~66

  • Working Set W(t, Δ): 시간 [t-Δ, t] 동안 참조한 페이지들의 집합 (Δ = window size)
  • 매순간 working set이 주기억장치에 있도록 유지 → thrashing 방지
  • 단점: 페이지 참조 시마다 상주 집합 조정 오버헤드 발생 (페이지 부재와 무관)
  • 추가 적재 없어도 교체될 수 있고, 추가 적재 있어도 교체 없을 수 있음

PFF (Page Fault Frequency) 기법 슬라이드 68~72

  • 페이지 부재율(inter-fault time, IFT)에 따라 할당량 조정
  • IFT > τ (임계값): 페이지 부재율 낮음 → 할당 줄임 (시간 tc-1~tc 참조 페이지만 유지)
  • IFT ≤ τ: 페이지 부재율 높음 → 할당 늘림 (기존 상주 집합 유지 + 새 페이지 추가)
  • 장점: 페이지 부재 발생 시에만 작동 → WS보다 오버헤드 적음

VMIN (Variable MIN) 기법 슬라이드 73~77

  • 가변 할당 기반 최적 기법 (페이지 부재 횟수 + 평균 할당량 모두 최소화)
  • 미래 참조 스트링이 필요 → 실현 불가능, 성능 평가 기준으로 사용
  • 최적 Δ = R/U (R: 페이지 부재 서비스 비용, U: 한 번의 참조 동안 한 페이지 유지 비용)

기타 고려사항 PPT 11 - 슬라이드 78~82

페이지 크기

  • 일반적 크기: 2⁷ ~ 2¹⁴ bytes
  • 작게 할 때: 테이블 크기 증가, 내부 단편화 감소, 필요한 부분만 적재 가능
  • 크게 할 때: 불필요한 부분 적재 → 낭비, 전송 횟수 감소

프로그램 구조 재구성

  • 시스템의 페이지 크기, 배열 저장 형태를 알고 프로그램 구조를 재구성하면 페이지 부재를 크게 줄일 수 있음
  • 예: C언어 행우선 저장 → 열 방향 반복 루프 대신 행 방향 반복 루프 사용

파일 시스템 논리적 구조 PPT 12 - 슬라이드 12~24

구조설명특징
평면 디렉토리전체에 하나의 디렉토리모든 파일 이름이 달라야 함, 다중 사용자 환경에서 불편
2-단계 디렉토리각 사용자마다 하나의 디렉토리하부 디렉토리 생성 불가
계층 디렉토리트리 구조, 하부 디렉토리 동적 생성/삭제대부분의 OS 채택, 루트/현재/홈 디렉토리 개념
비순환 그래프 디렉토리계층 구조 + 링크(link)로 공유하나의 파일을 여러 디렉토리에서 접근 가능
일반 그래프 디렉토리사이클(cycle) 존재 가능비순환 그래프의 일반화, 무한 순환 탐색 가능성 고려 필요

파일 시스템 물리적 구조 PPT 12 - 슬라이드 25~30

디스크 빈 공간 관리 기법 슬라이드 27~28

  • 패스워드 기법: 비현실적

접근 행렬(Access Matrix) 구현 기법 슬라이드 35~39

기법표현 방법장단점
전역 테이블<도메인명, 객체명, 권한집합> 순서 세쌍테이블 크기 커짐, 그룹화 어려움
접근 리스트(Access list)열(column) 단위: 객체별로 도메인과 권한 나열파일 생성시 권한 부여에 적합, 실제로 많이 사용 (UNIX: rwx rwx rwx)
권한 리스트(Capability list)행(row) 단위: 도메인별로 파일과 권한 나열프로세스가 스스로 권한 제시, 권한 리스트 자체 보호 필요
락-키 기법접근 리스트 + 권한 리스트 혼합객체에 lock, 도메인에 key
대부분의 OS에서 접근 리스트와 권한 리스트 개념을 혼합 사용 (UNIX도 마찬가지)

디스크 스케줄링 기본 PPT 13 - 슬라이드 3~5

탐구 시간 최적화 기법 PPT 13 - 슬라이드 6~24

기법방법장점단점
FCFS도착 순서대로 서비스공평, 오버헤드 적음, 부하 작을 때 적합부하 클 때 비효율
SSTF현재 헤드 위치에서 가장 가까운 요구 먼저처리량 극대화, 평균 응답 시간 낮음예측성 저하, 무기한 연기 가능
SCAN헤드 진행 방향으로 가장 가까운 요구 처리, 끝에서 방향 전환처리량·응답 시간 우수, 실제 스케줄링의 근간양 끝 실린더는 방향 전환 후 처리
LOOKSCAN과 동일, 단 더 이상 요구 없으면 끝까지 가지 않고 방향 전환 (Elevator algorithm)불필요한 헤드 이동 없음-
N-step SCAN방향 전환 시점의 큐 요구만 처리, 서비스 중 도착 요구는 다음 번에응답 시간 분산 작음, 무기한 연기 완전 배제-
C-SCAN한 방향으로만 요구 처리, 반대 방향 이동 시는 서비스 없음응답 시간 분산 매우 작음, 예측성 매우 높음이동 거리가 SCAN보다 길 수 있음
C-LOOKC-SCAN과 동일, 더 이상 요구 없으면 즉시 방향 전환불필요한 이동 없음-
SCAN/LOOK 계열이 일반적으로 많이 사용됨 (일반적인 부하 상태에서 적절한 성능)

회전 지연 시간 최적화 PPT 13 - 슬라이드 25~27

  • 고정 헤드 디스크(fixed head disk) 또는 같은 실린더에 여러 요구 있을 때 사용
  • 섹터 큐잉(Sector Queueing, SLTF): 각 섹터별 별도 큐, 헤드 아래 먼저 도착하는 섹터 순서대로 처리 → 거의 최적

디스크 스트라이핑과 RAID PPT 13 - 슬라이드 30~35

디스크 스트라이핑(Disk Striping) 슬라이드 30~31

  • 여러 개의 디스크를 논리적으로 하나의 디스크처럼 사용
  • 하나의 블럭을 여러 서브블럭으로 나누어 각 디스크에 배치
  • 입출력을 공평하게 분배, 블럭 접근 시간 단축, 병렬 접근 효과

RAID (Redundant Array of Inexpensive Disks) 슬라이드 32~35

RAID 수준특징
RAID 0디스크 스트라이핑, 낮은 비용·높은 성능, 신뢰성 없음
RAID 1미러링(mirroring), 높은 데이터 가용성, 읽기 성능 우수, 실제로 많이 사용
RAID 2섹터 스트라이핑 + ECC 별도 저장, 잘 사용 안 됨
RAID 3섹터 스트라이핑 + 한 디스크에 패리티 저장, 단일 태스킹 시스템에 사용
RAID 4패리티 디스크 고정, 트랜잭션 처리에 적합, 병목현상 발생, 잘 사용 안 됨
RAID 5회전 패리티 배열, 병목현상 억제, 다중 사용자 환경에 적합, 디스크 고장시 복구 가능

각 조건 제거 방법: 공유화 / 선점 지원(비실현적) / 전체 할당 기법(자원 낭비, 무기한 연기) / 자원 순서화(자원 낭비)
결론: 대부분의 경우 자원의 심각한 낭비 초래 → 현실적으로 사용하기 어려움
2 교착상태 회피 기법에서 안전상태와 비안전상태를 정의하고, Dijkstra 알고리즘과 Habermann 알고리즘의 차이를 설명하시오. 8장
안전상태: 모든 프로세스를 유한 시간 내에 정상 종료시킬 수 있는 상태 (safe sequence 존재)
비안전상태: safe sequence 없음. 교착상태로 전이될 가능성 있음 (반드시 교착상태가 발생한다는 의미는 아님)
Dijkstra: 자원형 한 가지, 자원 요청 시 할당 후 안전상태면 승인/비안전상태면 거절
Habermann: 여러 종류의 자원형, Dijkstra의 확장
가정: 프로세스 수·자원 수 고정, 최대 요구량 사전에 알려져야 함 → 비현실적
3 교착상태 검출 기법에서 자원 할당 그래프의 그래프 단순화(graph reduction) 과정을 설명하시오. 8장
unblocked 프로세스: 현재 상태에서 요구한 모든 자원을 할당받을 수 있는 프로세스
절차: ① unblocked 프로세스 찾기 → ② 인접한 모든 에지 제거 → ③ 더 이상 unblocked 프로세스가 없을 때까지 반복
결과: 모든 에지 제거 → 교착상태 없음 / 남은 에지 있음 → 교착상태 존재
단순화 순서에 관계없이 최종 그래프는 항상 동일 (정리)
4 고정 분할(FPM)과 가변 분할(VPM) 다중프로그래밍의 차이점을 내부/외부 단편화 개념과 함께 설명하시오. 9장
FPM: 미리 고정 크기로 분할 → OS 관리 용이 / 내부 단편화(분할 영역 내 낭비), 외부 단편화(크기 문제로 영역 전체 낭비) 발생
VPM: 프로세스 크기에 맞게 동적 분할 → 내부 단편화 없음 / 관리 오버헤드 증가, 인접 공간 통합·기억장소 통합 필요
배치 기법: first-fit, best-fit, worst-fit, next-fit 비교 설명
5 페이징 시스템에서 직접 사상, 연관 사상, 혼합 사상 기법의 차이점과 각각의 장단점을 설명하시오. 10장
직접 사상: PMT를 주기억장치 커널 공간에 저장, 주기억장치 접근 2회 필요 → 성능 저하
연관 사상: PMT를 연관 기억장치(content addressable memory)에 저장, 병렬 탐색 → 하드웨어 비용 큼
혼합 사상: 소규모 연관 기억장치 + 주기억장치에 전체 PMT. 최근 사용 페이지만 연관 기억장치에 유지 → 지역성 활용
혼합 사상 알고리즘: 연관 기억장치에 있으면 바로 사용, 없으면 직접 사상 + 연관 기억장치에 적재
6 페이징 시스템과 세그먼테이션 시스템의 특징을 비교하고, 혼합 기법의 주소 사상 과정을 설명하시오. 10장
페이징: 같은 크기 블럭, 단순·효율적, 논리적 분할 없음, 공유 복잡
세그먼테이션: 다른 크기의 논리적 블럭, 공유·보호 쉬움, 관리 오버헤드 증가
혼합: 가상 주소 v = (s, p, d), SMT → 세그먼트의 PMT 주소 인출 → PMT → 페이지 프레임 번호 인출 → 실 주소 계산
SMT에 존재 비트 없는 이유: 세그먼트 단위가 아닌 페이지 단위로 적재되기 때문
7 FIFO 알고리즘의 Belady's Anomaly(FIFO Anomaly)를 예를 들어 설명하고, LRU 알고리즘과 비교하시오. 11장
FIFO Anomaly: FIFO 알고리즘에서 주기억장치 할당량을 늘려주었는데 페이지 부재 발생 횟수가 오히려 증가하는 경우 (예: ω = 1 2 3 4 1 2 5 1 2 3 4 5에서 프레임 3개 → 9회, 4개 → 10회)
LRU: 최근 가장 오래 참조되지 않은 페이지 교체, 지역성 기반, MIN에 근사, 실제 가장 많이 사용
LRU 구현: 카운터 방법(참조 시간 기록) 또는 스택 방법(최근 참조 순서 유지)
LRU 단점: 참조 시간 기록 오버헤드, 대형 반복 구조에서 성능 급격 저하
8 NUR 알고리즘과 Clock 알고리즘의 동작 원리를 참조 비트와 갱신 비트를 중심으로 설명하시오. 11장
NUR: 참조 비트(r), 갱신 비트(m) 이용. 교체 우선순위: (0,0)→(0,1)→(1,0)→(1,1). LRU 오버헤드를 줄이면서 LRU 성능 근사
Clock: 환형 리스트의 포인터를 시계 방향으로 이동, 참조 비트=0이면 교체, 1이면 0으로 설정 후 진행. 참조 비트를 주기적으로 재설정하지 않음. IBM VM/370 사용
Second Chance: Clock + 갱신 비트. (0,0)→교체, (0,1)→소거 후 진행, (1,0)→(0,0)후 진행, (1,1)→(0,1)후 진행
9 Working Set(WS) 기법과 PFF 기법을 비교하고, 각각의 장단점을 설명하시오. 11장
WS: W(t, Δ) = 시간 [t-Δ, t] 동안 참조한 페이지 집합. 매순간 상주 집합 조정 필요 → 페이지 부재와 무관하게 오버헤드 발생. thrashing 방지 효과
PFF: inter-fault time(IFT)이 임계값 τ보다 크면 할당 줄임, 작으면 할당 늘림. 페이지 부재 발생시에만 동작 → WS보다 오버헤드 작음
성능 평가: 단순히 페이지 부재 횟수만이 아닌 평균 주기억장치 할당량도 함께 고려해야 함
10 파일 시스템의 논리적 구조 5가지를 설명하고, 계층 디렉토리 구조의 특징을 서술하시오. 12장
평면 → 2단계 → 계층 → 비순환 그래프 → 일반 그래프 순서대로 발전
계층 디렉토리: 트리 구조, 하부 디렉토리 동적 생성/삭제, 대부분의 OS 채택
루트 디렉토리(최상위), 현재 디렉토리(default 접근), 홈 디렉토리(로그인 시 설정)
비순환 그래프: 링크(link) 개념으로 파일 공유, 계층 디렉토리의 확장
일반 그래프: 사이클 존재 가능, 무한 순환 탐색 주의
11 파일 보호를 위한 접근 행렬의 구현 기법인 접근 리스트와 권한 리스트를 비교하고, UNIX 운영체제에서의 파일 보호 기법을 설명하시오. 12장
접근 리스트(Access list): 행렬의 열 단위, 객체(파일)별로 도메인과 권한 나열. 파일 생성 시 권한 부여 용이, 실제 많이 사용
권한 리스트(Capability list): 행렬의 행 단위, 도메인별로 파일과 권한 나열. 프로세스가 스스로 권한 제시, 권한 리스트 보호 필요
UNIX: rwx rwx rwx (소유자/그룹/기타 각 R, W, X 권한). 접근 리스트와 권한 리스트 개념 모두 사용
12 디스크 스케줄링의 SCAN, N-step SCAN, C-SCAN 기법의 차이점을 설명하고, 각 기법의 응답 시간 예측성 측면에서 비교하시오. 13장
SCAN: 진행 방향으로 가장 가까운 요구 서비스, 끝에서 방향 전환. 서비스 중 도착한 요구도 같이 처리 가능. 무기한 연기 가능성
N-step SCAN: 방향 전환 시점 큐에 있는 요구만 처리, 중간 도착 요구는 다음 회전 때 처리. 응답 시간 분산 작음, 무기한 연기 완전 배제
C-SCAN: 미리 정해진 방향으로만 서비스, 반대 방향 이동 시 서비스 없음. 응답 시간 분산 매우 작음, 예측성 매우 높음
일반적으로 SCAN/LOOK 계열이 많이 사용됨
13 RAID 0, 1, 5의 특징을 비교하고, 각각 어떤 상황에서 사용하기 적합한지 설명하시오. 13장
RAID 0: 스트라이핑, 낮은 비용·높은 성능, 신뢰성 없음 → 성능만 중요할 때
RAID 1: 미러링, 높은 데이터 가용성, 읽기 성능 우수 → 신뢰성이 중요한 환경, 실제로 많이 사용
RAID 5: 회전 패리티 배열, 병목현상 억제, 디스크 고장 복구 가능, 다중 사용자 환경에 적합 → 성능·신뢰성·비용 균형
CAP 삼각형(Cost-Availability-Performance) 관점에서 각 수준의 위치
14 가상기억장치 관리에서 스래싱(thrashing) 현상이 무엇이며, 이를 방지하기 위한 기법을 설명하시오. 11장
스래싱: 다중프로그래밍 정도가 너무 높아 페이지 부재가 과도하게 발생하는 현상 → 시스템 성능 급격 저하, CPU 이용률 감소
방지 기법:
1. 부하 조정 기법: 다중프로그래밍 정도를 plateau 영역에서 적절히 유지
2. Working Set 기법: 매순간 Working Set이 주기억장치에 있도록 유지
3. PFF 기법: 페이지 부재율에 따라 할당량 동적 조정
4. 가변 할당 기법 전반: 프로세스 특성에 맞게 할당량 조정
15 교착상태 복구 기법인 프로세스 종료 기법과 자원 선점 기법을 비교하고, 검사점-재시작 기법을 설명하시오. 8장
프로세스 종료 기법: 교착상태 프로세스 중 일부 강제 종료 후 재시작. 종료 비용 산출(우선순위, 실행 시간, 남은 시간 등)
- lowest-termination-cost first: 단순, 오버헤드 적음, 불필요한 종료 가능
- minimum cost recovery: 최소 비용, 선정 과정 복잡
자원 선점 기법: 교착상태 제거를 위해 선점할 자원 선정, 해당 자원 보유 프로세스를 종료 후 재실행. 교착상태 아닌 프로세스도 종료 대상이 될 수 있음
검사점-재시작: 검사점에서 프로세스 context 저장, 강제 종료 후 가장 최근 검사점부터 재실행 → 재실행 비용 최소화
