문항 간 상관행렬 — 만족도조사 (17문항)

색상 기준 (Pearson r)
r ≥ 0.60 (높은 정적 상관) 0.40~0.60 (중간 상관) 0.20~0.40 (낮은 상관) |r| < 0.20 (무상관)
해석
영역 내 상관 (동일 영역 문항끼리): r = 0.49~0.61 — 중간 이상의 정적 상관. 같은 영역의 문항들이 동일한 구인을 측정하고 있음을 나타냄.
영역 간 상관 (서로 다른 영역): r ≈ −0.11~+0.19 — 실질적 무상관. 5개 영역이 독립적인 구인으로 분리되어 있음을 확인.
종합: 블록 대각 구조(5개 색상 블록이 대각선 방향으로 집중) → EFA 5요인 구조를 지지하는 패턴.
결론: 문항 간 상관행렬에서 영역별 문항의 상관을 확인한 결과, 동일 영역 내 문항끼리는 높은 상관(r=0.49~0.61)을 보이고 다른 영역 문항과는 낮은 상관(r≈0)을 보임. 이를 통해 영역별로 문항이 잘 묶여 있음을 확인할 수 있음.

타당도 분석 — 만족도조사

타당도(Validity)란?
  • 측정 도구가 측정하려고 의도하는 것을 얼마나 충실히 측정하는지
  • 측정 도구가 무엇을 측정하고 있는지에 대한 설명이나 지수
  • 0/1 이분법이 아닌 정도의 개념 — "낮다 · 적절하다 · 높다"로 표현
구인 타당도 (Construct Validity) 중점 분석
내적 구조에 기초한 타당도
"검사가 측정하려는 심리적 구인(요인)을 올바르게 측정하는가?"
탐색적 요인분석 (EFA)
회전: Varimax · 추출 요인: 5개 · 방법: minres
→ 설문 5개 영역의 이론적 구인과 실제 요인 구조 일치
분석 결과: 5개의 요인이 도출되었으며, 요인 부하량 기준 0.40 이상으로 잡았을 때 각 문항이 해당 영역의 요인에만 높게 적재되고 나머지 요인에는 낮은 값을 보임. 문항이 요인별로 잘 묶인 것을 확인할 수 있음.
결론: 5요인이 교육내용·강사·운영시설·취업지원·전반만족 각 영역에 완벽히 대응. 구인 타당도 확보.

Wright Map — 피험자 역량 분포 vs 문항 난이도

Wright Map이란?
  • 피험자 역량과 문항 난이도를 같은 로짓 척도에 배치
  • 역량이 높을수록 위쪽, 난이도가 높을수록 위쪽
  • 동일 로짓 위치의 피험자가 해당 문항을 정답률 50%로 맞힘
변별도 확보 근거
상 난이도 logit ≈ +0.65 → 역량 상위 피험자만 정답
총점 높은 그룹이 맞히고, 낮은 그룹은 틀림
중 난이도 logit ≈ −0.07~+0.14 → 중간 역량 기준 변별
하 난이도 logit ≈ −0.74 → 역량 하위도 대부분 정답
기저 변별 역할
결론: 총점이 높은 피험자가 난이도 높은 문항을 정답 → 문항이 역량 수준을 잘 변별함.

팀원 소감

윤재원 2024100782

이번 HRD 프로젝트를 통해 그동안 이론으로만 알고 있던 HRD의 개념들을 직접 탐구하며 훨씬 깊이 있게 이해할 수 있었습니다. 팀원들과 함께 고민하고 의견을 나누는 과정이 생각보다 즐거웠고, 협업 속에서 자연스럽게 흥미와 동기를 얻을 수 있었습니다.

또한 프로젝트를 진행하는 동안 Claude와 같은 AI 도구를 실제로 활용해보면서, 도구를 효과적으로 사용하는 방법에 대한 감각도 키울 수 있었습니다. 정보를 찾고 정리하는 방식 자체가 달라진 것 같아 뿌듯함을 느꼈습니다.

장은비 2024100158

이번 프로젝트에서 더미데이터 생성 규칙과 분석설계를 작성해보면서, Claude를 통해 설계의 명확성이 얼마나 중요한지 느꼈습니다. 데이터 규칙을 정의하고 분석 방법을 기획하는 과정에서 이론으로만 배우던 신뢰도, 타당도, 집단비교 같은 개념들을 직접 적용해볼 수 있었습니다. 산출물을 검토하면서 설계 의도가 실제 분석 결과에 어떻게 반영되는지 확인했고, 그 과정에서 데이터 분석에 대한 이해가 한 단계 깊어진 것 같습니다. 팀원들과 함께 산출물을 검토하면서 문항의 문제를 찾아내고 차트를 개선해야 한다는 것을 확인했습니다. 설계 의도와 실제 결과를 비교하고 조정하는 과정이 필요했고, 그러한 피드백 과정을 통해 더 정교한 분석이 이루어질 수 있다는 것을 배웠습니다.

이선학 2024100761

관련 분야에 대한 지식이 충분하지 않은 상태에서 AI를 활용하다 보면, AI가 조금만 앞서 나가도 그 결과물이 맞는지 틀린지 판단하기 어려운 순간이 찾아온다. 이번 작업을 통해 AI와의 협업에서는 작은 단계마다 승인을 받는 Human-in-the-loop가 단순한 형식이 아니라 실질적인 품질 관리 장치임을 체감했다. 검증 없이 몇 단계를 넘어가면 방향 자체가 틀어지기 쉽고, 그것을 뒤늦게 되돌리는 비용이 훨씬 크다는 것도 깨달았다.

그럼에도 이번 작업 전반은 매우 의미 있는 경험이었다. AI를 활용해 평가 문항을 설계하고, 더미데이터를 생성한 뒤, 평가 도구 자체의 질(신뢰도·타당도)과 성취 수준을 동시에 분석하는 파이프라인을 직접 구축하고 검증해 본 경험은 단순한 결과물 이상의 가치를 지닌다. CTT 분석의 흐름을 실제 데이터를 통해 체득했다는 점에서 스스로에게 가장 값진 성과였다.

유용주 2024100023

이번 프로젝트는 설문 설계와 성취도 설계를 Claude Code를 이용해 구현했다. 하네스 설계로 파이프라인을 만들면 각 단계를 확인하며 진행할 수 있었고, 설계를 의도한 대로 명확하게 작성하는 과정이 중요하다고 느꼈다. 설문 문항과 성취도 문항을 만들고 더미 데이터를 활용한 분석 과정을 검토하며 이론 내용을 복습할 수 있었다.
